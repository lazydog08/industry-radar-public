# TASK-06 移动端 UI 和阅读体验优化

- 更新时间：2026-05-26 14:45 CST
- 负责范围：`src/web/styles.css` 为主；未改数据加载逻辑，未改 API。

## 优化目标

让首页在 iPhone 竖屏下更像“每天刷一眼的行业情报”，优先保证快速判断、纵向阅读、可点击溯源和评分含义清楚。

## 已完成调整

- 首屏信息密度：手机端概览指标改为 2 列，减少一屏内纵向占用。
- 一图读懂：手机端降低内边距和固定高度，三段内容改为单列卡片，评分条更窄，避免标题和正文挤压。
- 评分体系：保留 `A 重点 / B 观察 / C 背景 / D 暂存` 彩色胶囊，手机端图例压缩为 2 列，隐藏次级提示，先让等级含义可扫读。
- 来源链接：一图读懂底部继续使用短来源标签，移动端限制域名宽度；知识卡保留完整可点击来源，长 URL 在手机端折叠显示，避免撑破版面。
- 筛选区：手机端改为“关键词 + 搜索”优先，分类/来源/实体/标签和反馈筛选降为更紧凑布局，减少对首屏阅读流的干扰。
- 知识卡：正文行高、段落间距、评分卡和操作按钮在手机端重新压缩，适配 200-300 字知识卡阅读。

## 验证方法

- 静态检查：`node --check src/web/app.js`
- 类型检查：`pnpm typecheck`
- 浏览器验证：使用本地页面 `http://localhost:3887/`，分别检查 390px 手机宽度和 1200px 桌面宽度。

## 集成注意

- 本任务只改样式和文档，未触碰数据抓取、静态导出或线上只读模式。
- 若 TASK-02 后续改动页面结构，请保留以下语义 class，移动端样式会继续生效：`infographic`、`score-pill`、`source-chip`、`filters`、`event-card`、`knowledge-fields`。
- 如果线上静态页面需要隐藏反馈操作，可以在 TASK-02 中增加只读 class，再由本任务样式追加处理。

## 遗留风险

- 当前筛选区仍是完整控件，只是移动端压缩排布；如果后续希望首屏更像纯阅读产品，建议把高级筛选改成可展开面板。
- 还没有引入真实截图回归测试；建议最终集成阶段补 390px 和 1200px 的截图基线。
