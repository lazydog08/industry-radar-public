# Radar Score 评分系统复查

- 日期：2026-05-26
- 任务：TASK-07 评分系统复查与优化
- 目标：让评分更直接回答“这条情报值不值得我今天看、继续跟，或转成视频选题”。

## 结论

Radar Score 应该优先衡量“今日处理优先级”，而不是单纯衡量“能不能做视频”。原因是视频选题价值经常来自二次加工，一条旧闻或单来源争议也可能有选题角度，但不应该因此挤占首页高优先级。

本次调整后：

- 热度趋势权重从 20 提高到 26，成为单项最高权重。
- 变化强度权重从 16 降到 10，避免“有发布/更新字样”但没人讨论的普通新闻虚高。
- 视频潜力继续保留为独立的 1-5 分字段，不直接并入 100 分总分。
- 低可信、旧闻、mock/sample/example 来源继续封顶，避免假热点进入首页。

## 新权重

| 维度 | 权重 | 作用 |
| --- | ---: | --- |
| 相关度 | 22 | 是否命中数码、AI、汽车、平台生态，以及用户重点关注品牌/平台 |
| 热度趋势 | 26 | 是否有热度、跨源扩散、热议/首发/开放/调整等扩散信号 |
| 新鲜度 | 18 | 是否是今天或近几天刚出现的信息 |
| 变化强度 | 10 | 是否存在明确规则、发布、OTA、价格、开售、限制等变化 |
| 可信度 | 16 | 是否来自官网、IT之家、Android/Apple 官方等较可信来源，是否多源验证 |
| 稀缺性 | 8 | 是否还没有被过度转发，是否存在早期观察价值 |

总分保持 100 分，继续输出已有字段：

- `radar_score`
- `radar_level`
- `radar_section`
- `video_potential`
- `confidence`
- `freshness_label`
- `score_parts`
- `caps`

## 视频创作用法

系统现在把两个问题分开：

1. `radar_score`：今天要不要优先处理。
2. `video_potential`：它有没有可能被加工成视频。

因此，一条内容可能出现这些情况：

- 高 Radar + 高视频潜力：优先看，马上判断要不要做。
- 高 Radar + 低视频潜力：行业重要，但可能只适合做背景判断。
- 低 Radar + 高视频潜力：有角度，但不该自动进首页，适合放入备选池。
- 低 Radar + 低视频潜力：仅归档。

## 分区逻辑

- `must_read`：新鲜且总分高，适合优先阅读。
- `developing`：有趋势或事件变化，但还需要继续验证。
- `video_ready`：视频潜力高，同时总分不低且置信度不能是低置信。
- `background`：旧闻、时间不明、低分、低可信或仅适合沉淀。

这次降低了视频潜力对首页分区的影响：低分或低置信内容即使视频潜力高，也不会只因为“好做视频”就被推到 `video_ready`。

## 封顶规则

以下规则继续保留：

- 30 天以上旧内容最高 35 分。
- 发布时间不明确最高 55 分。
- Mock / sample / example 来源最高 45 分。
- 单一弱来源或单一社交来源最高 60 分。

这些封顶比权重更优先，目的是防止旧闻、示例数据、单点传言成为高优先级内容。

## 10 条样例评分解释

| 样例 | 预期评级 | 解释 |
| --- | --- | --- |
| OPPO 发布会官宣，多个科技媒体跟进 | A 重点 | 命中数码/影像/发布会，信息新，多源扩散，可信度较高，适合优先判断是否做发布会前瞻。 |
| 华为 FreeClip 2 与 nova 16 同台发布消息 | A/B | 命中华为和新品发布，若来自可靠媒体且当天扩散，可进入重点或观察；若只有单源则保持观察。 |
| B站创作激励和流量规则变化 | B 观察 | 与平台生态和创作者高度相关，但需要看是否有官方口径或多源确认；单一社交来源会封顶。 |
| 特斯拉超级充电卡开放或价格变化 | B 观察 | 与汽车和补能相关，有用户价值和视频角度；如果热度扩散不足，不直接进 A。 |
| 小米汽车智驾 OTA 或安全规则更新 | A/B | 命中汽车、智驾、OTA，变化明确；若多源且当天热议可 A，否则 B。 |
| Apple / Android 官方系统规则调整 | A 重点 | 官方来源可信，规则变化会影响用户和内容解读，适合优先看。 |
| 手机壳、配色、联名周边上新 | D 暂存 | 命中低价值关键词，即使新鲜也不应该占据情报首页。 |
| 30 天前的手机爆料重新被转发 | D/C | 旧内容封顶，除非出现新的官方确认或多源反转，否则只作背景。 |
| 单一知乎/微博争议帖爆火 | B/C | 可能有趋势，但低可信和单一社交来源会封顶；适合补来源后再判断。 |
| 供应链新技术首次被少数可靠来源报道 | B 观察 | 稀缺性高但可能热度不足，适合观察，不急于推成今日必看。 |

## 给前端和集成的说明

- 前端评分条最大值需要使用：相关度 22、趋势 26、新鲜度 18、变化 10、可信度 16、稀缺性 8。
- 不要把 `video_potential` 当作 Radar Score 的一部分展示。
- 推荐文案应优先解释“为什么今天值得看/跟进”，再解释“能不能做视频”。
- 集成时如果新增静态导出，必须保留 `score_parts` 和 `caps`，否则线上页面无法解释评分。

## 遗留风险

- `heatScore` 不同来源的口径不统一，趋势分仍然需要靠真实抓取结果继续校准。
- 知乎、微博公开访问受限时，社交热度会偏低，需要后续接入更稳定的公开趋势源或人工补来源。
- 当前样例是规则解释，不代表真实事件固定分数；真实分数仍取决于来源数量、发布时间、热度和封顶规则。
